## 2. デザインルール

### 2.0 最重要原則：強弱（メリハリ）のあるデザイン

本サイトのデザインは **視覚的な強弱を明確につける** ことを最重要原則とする。均一で平坦な見た目は避け、以下の観点で常にコントラストを意識すること。

| 観点 | 「強」（目立たせる） | 「弱」（控えめにする） |
|------|--------------------|-----------------------|
| テキスト | 見出し: `#ffffff`, weight 800〜900, 大サイズ | 補足: `#6b6b80`, weight 400〜500, 小サイズ |
| 余白 | ヒーロー・セクション間は大きく空ける | セクション内の要素間は詰めてグルーピングする |
| 色 | アクセント（`#00d4ff` `#7c3aed`）で注目点を強調 | 背景は極暗（`#06060b`）に沈めて主役を引き立てる |
| アニメーション | ホバー・クリック応答は素早く・はっきり | 背景アニメーションはゆっくり・控えめ |
| 装飾 | セクション境界にグラデーションライン、ホバーでグロー | 通常状態のボーダーは薄く・低コントラスト |

**アンチパターン（避けるべきこと）:**
- 全要素が同じフォントサイズ・同じ太さ・同じ色
- 背景とカードの色差がほとんどない
- ホバーしても変化がわずかで気づきにくい
- 余白が均一でセクション境界が不明瞭
- アニメーションが全要素で同タイミング・同速度

### 2.1.1 テーマアーキテクチャ

- テーマ切り替えは `html[data-theme="light"]` / `html[data-theme="dark"]` でCSS変数をオーバーライドする方式
- `:root` にダークテーマのデフォルト値を定義し、`html[data-theme="light"]` で全変数をオーバーライド
- テーマ状態は `localStorage('theme')` で永続化、未設定時は `prefers-color-scheme` に追従
- FOUC防止のため `<head>` 内の `<link rel="stylesheet">` 前にブロッキングスクリプトを配置
- テーマ切替時は `.theme-transitioning` クラスで0.3sの遷移アニメーション
- **新しい色値は必ずCSS変数として定義し、両テーマのパレットを設定すること**
- SVG画像は `<img>` タグで読み込んでいるためCSS変数の影響を受けない（変更不要）

### 2.3 アニメーション・モーション

| 種類 | 内容 | 速度・挙動 |
|------|------|-----------|
| ヒーローグロー | 集中グローの呼吸（拡縮+透明度変化） | 10s / ease-in-out / 無限ループ |
| 背景グリッド | 半透明グリッドパターンのスクロール | 12s / linear / 無限ループ |
| 背景グラデーション | `body::before` のパルス（拡縮＋透明度変化） | 8s / ease-in-out / 無限ループ |
| ヒーロー文字グラデーション | アクセント文字の色がシフト | 5s / ease / 無限ループ |
| カードホバー | 浮き上がり + グロー + ボーダー発光 | 0.35s / cubic-bezier(0.22, 1, 0.36, 1) |
| サムネイルホバー | scale(1.08) ズーム | 0.5s / cubic-bezier(0.22, 1, 0.36, 1) |
| スクロール登場 | fade-in-up（下から浮き上がり） | 0.7s / IntersectionObserver / stagger 0.1s刻み |
| ナビリンク | アンダーラインが左から伸びる | 0.3s / ease |

**`prefers-reduced-motion: reduce` を必ず尊重する** — アニメーション無効時はすべて即座に完了状態にする。

### 2.6 コンポーネント

#### ロゴ
- テキストベースのタイポグラフィロゴ（SVGアイコンなし）
- 「V」は `.logo-v` クラスで `font-weight: 900; font-size: 1.15em` — 太字で少し大きく強調
- 「q」は `.logo-q` クラスで `linear-gradient(135deg, accent-primary, accent-secondary)` のグラデーション文字
- ヘッダー・フッター全ページ統一
- ホバー時に `drop-shadow` グロー

#### ヘッダー
- 固定ヘッダー（sticky）
- 背景: `var(--header-bg)` + `backdrop-filter: blur(20px) saturate(1.5)`
- SVGロゴ + サイト名（ホバーでグロー） + ナビリンク（ホバーでアンダーラインアニメーション）
- テーマトグルボタン（`.theme-toggle`）: ナビ右端に配置、ダーク時は太陽アイコン、ライト時は月アイコン

#### ヒーロー
- ラベルバッジ（ボーダーがパルスアニメーション）
- 見出し（グラデーションアニメーション付きアクセント文字）
- 背景: 集中グロー + アクセントライン + グレイン（`.hero-geo`）
  - 集中グロー（`hero-glow`）: dual-color radial gradient（cyan + purple）、`blur(60px)`、`--hero-glow-*` CSS変数でテーマ追従、ゆっくり呼吸アニメーション
  - ビネット（`hero-geo::after`）: 周辺を暗くして中心に視線を集中
  - アクセントライン（`hero-accent-line`）: シャープな1pxグラデーション + `::before`ブルームハロー。柔らかいグローとの対比でメリハリを出す
  - フィルムグレイン（SVG `feTurbulence`、`mix-blend-mode: overlay`、`opacity: 0.04`）
- グラデーション区切り線（`hero-divider`）

#### 記事カード
- サムネイル画像（16:9、グラデーションオーバーレイ付き）
- カテゴリタグ（ボーダー付きバッジ）
- タイトル（h2、weight: 800）
- 概要文（2〜3行でclamp、color: `#6b6b80`）
- 公開日（最も控えめ: 小サイズ・低opacity）
- ホバーでカード浮き上がり + グロー + ボーダー発光

#### セクション見出し
- 左にグラデーションバー（`accent-primary → accent-secondary`）
- 大きく太いフォント

#### フッター
- 背景: `#0e0e18`
- 上辺にグラデーションライン（`transparent → accent → transparent`）
- コピーライト + サイトリンク

## 3. ライティングルール

### 3.1 文体・語調

- **基本文体**: です・ます調
- **トーン**: 信頼感があり、フレンドリーだが軽すぎない
- **視点**: 読者に語りかける二人称（「あなた」は控えめに使用）

### 3.2 専門用語

- 初出時に簡潔な説明を括弧内に添える
  - 例: 「LLM（大規模言語モデル）は〜」
- 広く知られた略語（HTML, CSS, APIなど）は説明不要
- 判断に迷ったら補足をつける

### 3.3 文の構成

- **1文の長さ**: 60文字以内を目安に
- **1段落**: 3〜5文程度
- **見出し**: 具体的で内容が想像できるものにする

### 3.4 記事構成

```
1. 導入（リード文）: 何の記事か、読むメリットは何か（2〜3文）
2. 背景・概要: テーマの前提知識や背景情報
3. 本論: メインコンテンツ（h2で区切り、h3で詳細化）
4. 実践・具体例: コード例、手順、比較表など
5. まとめ: 要点の整理、今後の展望
```

### 3.5 推奨表現

- 「〜できます」「〜が可能です」
- 「具体的には」「例えば」
- 「注意が必要です」「確認しましょう」
- 「〜とは、〜のことです」（専門用語の補足）

### 3.6 NG表現

| NG | 推奨 |
|----|------|
| 絶対に / 必ず | 基本的に / 多くの場合 |
| すごい / やばい | 注目すべき / 重要な |
| 〜してください（連発） | バリエーションを持たせる |
| こちら（リンクテキスト） | 具体的な内容をテキストにする |

## 4. SEOルール

- **titleタグ**: 32〜54文字、キーワードを前方に配置
- **meta description**: 72〜120文字、行動喚起を含む
- **h1**: 各ページ1つのみ
- **見出し階層**: h2→h3→h4（飛ばさない）
- **画像alt属性**: 全画像に設定
- **内部リンク**: 記事内に関連記事リンクを1つ以上

---

## 5. ファイル命名規則

- **記事HTML**: `articles/YYYYMMDD-slug.html`
- **slug**: 英数字とハイフンのみ、簡潔に内容を表す
  - 例: `20260402-ai-code-review.html`
- **CSS**: `assets/css/style.css`（一元管理）
