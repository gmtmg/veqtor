# IMAGE_GUIDE.md — Veqtor 画像管理ガイド

このファイルは画像の命名規則・フォルダ構成・管理ルールを定義します。
image-manager および全エージェントはこのガイドに従ってください。

---

## 1. フォルダ構成

```
assets/images/
├── articles/      # 記事本文内で使用する画像
├── thumbnails/    # 記事カード用サムネイル画像
└── ui/            # サイトUI用の画像・アイコン
```

---

## 2. 命名規則

### 2.1 記事画像

**パターン**: `{YYYYMMDD}-{slug}-{連番}.{ext}`

| 要素 | 説明 | 例 |
|------|------|----|
| YYYYMMDD | 記事の公開日 | 20260402 |
| slug | 記事のスラッグ（記事ファイル名と一致） | ai-news |
| 連番 | 01から順に | 01, 02, 03 |
| ext | 拡張子 | jpg, png, webp, svg |

**例**:
- `assets/images/articles/20260402-ai-news-01.jpg`
- `assets/images/articles/20260402-ai-news-02.png`

### 2.2 サムネイル画像

**パターン**: `{YYYYMMDD}-{slug}-thumb.{ext}`

**例**:
- `assets/images/thumbnails/20260402-ai-news-thumb.jpg`

### 2.3 UI画像

**パターン**: `{用途名}.{ext}`

**例**:
- `assets/images/ui/logo.svg`
- `assets/images/ui/icon-search.svg`
- `assets/images/ui/hero-bg.jpg`

## 3. 推奨フォーマットとサイズ

### 3.1 フォーマット優先順位

| 用途 | 推奨フォーマット | 備考 |
|------|----------------|------|
| 写真・スクリーンショット | WebP > JPEG | WebPが最適、フォールバックにJPEG |
| 図解・ダイアグラム | SVG > PNG | ベクター形式を優先 |
| アイコン | SVG | スケーラブル |
| ロゴ | SVG | スケーラブル |

### 3.2 推奨サイズ

| 用途 | 推奨サイズ | アスペクト比 |
|------|-----------|------------|
| サムネイル | 800 x 450 px | 16:9 |
| 記事内画像 | 最大幅 1200px | — |
| OG画像 | 1200 x 630 px | 約1.91:1 |
| アイコン | 24 x 24 px / 32 x 32 px | 1:1 |

### 3.3 ファイルサイズ上限

| 用途 | 上限 |
|------|------|
| サムネイル | 200KB |
| 記事内画像 | 500KB |
| UI画像 | 100KB |

500KBを超える画像は最適化が必要。image-managerが警告を出す。

## 4. alt属性ルール

| 画像の種類 | alt属性の書き方 | 例 |
|-----------|---------------|-----|
| コンテンツ画像 | 画像の内容を具体的に記述 | `alt="GPT-4oのアーキテクチャ図"` |
| スクリーンショット | 画面の内容を記述 | `alt="VS Codeのデバッグ画面"` |
| グラフ・チャート | データの要約を記述 | `alt="2025年AI市場規模の推移グラフ"` |
| 装飾画像 | 空文字 | `alt=""` |
| アイコン | アイコンの意味を記述 | `alt="検索"` |

## 5. 整合性チェック

### 5.1 定期チェック項目

- [ ] 全画像ファイルが命名規則に従っているか
- [ ] 孤立画像（どの記事からも参照されていない画像）がないか
- [ ] ARTICLE_INDEX.mdに記載された記事に対応するサムネイルが存在するか
- [ ] alt属性が全画像に設定されているか
- [ ] ファイルサイズが上限を超えていないか

### 5.2 孤立画像の検出方法

```bash
# 画像フォルダの全ファイルをリスト
find assets/images -type f | sort

# HTMLから参照されている画像をリスト
grep -roh 'src="[^"]*images[^"]*"' articles/ index.html | sed 's/src="//;s/"//' | sort -u

# 差分を確認（孤立画像 = フォルダにあるがHTMLから参照されていない）
```

## 6. プレースホルダー画像

記事執筆時に画像が未準備の場合、HTMLにプレースホルダーコメントを記載する：

```html
<!-- IMAGE: [説明] | ファイル名: YYYYMMDD-slug-01.jpg | alt: [代替テキスト] -->
<img src="../assets/images/articles/YYYYMMDD-slug-01.jpg"
     alt="[代替テキスト]"
     loading="lazy">
```

image-managerが後から画像を配置する際の手がかりとなる。
